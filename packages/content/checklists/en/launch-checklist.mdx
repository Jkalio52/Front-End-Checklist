The Launch Checklist is the fast release gate for the fundamentals that most often cause public regressions. Use it before a new site, a major redesign, or any release where basic trust signals need to be verified before deployment.

## Who It's For

- Launch owners and release managers
- Junior developers who need a clear first pre-launch pass
- Teams that want a practical minimum before deeper audits

## Done Looks Like

You should finish this checklist knowing the site clears the baseline checks for document setup, crawlability, accessibility fundamentals, transport security, and basic performance hygiene.

## Why This Matters

Launching without these fundamentals can lead to:
- Poor search engine rankings
- Security vulnerabilities
- Broken user experiences
- Accessibility issues

## How to Use This Checklist

1. Go through each item systematically
2. Mark items as complete as you verify them
3. Use the "View details" link for implementation guidance
4. Treat failures as launch blockers unless the team explicitly accepts the risk

## Quick Tips

- Test on multiple devices before launch
- Verify all forms and interactive elements work
- Check that analytics and tracking are configured
- Have a rollback plan ready
