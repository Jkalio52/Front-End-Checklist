# Comparative Audit: Frontend Checklist vs squirrelscan

This folder holds results and instructions for comparing our `audit_url` tool against [squirrelscan](https://squirrelscan.com/) (and other audit tools) on the same set of URLs. The goal is to document **gaps**: where our 377-rule corpus finds issues that others miss, and vice versa.

## 1. Run our audit (Frontend Checklist)

**Prerequisites:** Dev server running (`pnpm dev` from repo root).

```bash
# From repo root; uses http://localhost:3000 by default
pnpm tsx scripts/compare-audit.ts
```

Optional: run against the deployed MCP server:

```bash
BASE_URL=https://mcp.frontendchecklist.io pnpm tsx scripts/compare-audit.ts
```

**Output:** `docs/audit-comparison/our-results.json` — one audit result per URL (summary + issues).

## 2. Run squirrelscan on the same URLs

Install squirrelscan (see [docs](https://docs.squirrelscan.com/quickstart)):

```bash
curl -fsSL https://squirrelscan.com/install | bash
```

Run a single-page audit for each URL (no crawl) for a fair comparison with our single-page `audit_url`:

```bash
# Example: one URL, JSON output
squirrel audit https://example.com -f json -o docs/audit-comparison/squirrel-example-com.json
```

To batch all 10 URLs (bash):

```bash
mkdir -p docs/audit-comparison/squirrel
for url in https://example.com https://web.dev https://developer.mozilla.org https://frontendchecklist.io https://www.w3.org https://nextjs.org https://react.dev https://vercel.com https://github.com https://stackoverflow.com; do
  name=$(echo "$url" | sed 's|https\?://||;s|\.|_|g;s|/|_|g')
  squirrel audit "$url" -C quick -f json -o "docs/audit-comparison/squirrel/${name}.json"
done
```

**Note:** squirrelscan crawls by default; use `-C quick` or `-m 1` to approximate single-page checks. Our tool only audits the given URL’s HTML.
